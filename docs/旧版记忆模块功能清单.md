# 旧版记忆模块功能清单

> 来源：`emet-memory/worker.js`（v6.8.2 旧前端，5287 行）内嵌前端的记忆模块完整实现。
> 用途：二期「能写」迁移对照表。每条前面的 `[ ]` 由静怡勾选：要迁移 / 要改 / 不要。
> 新前端已有的实现差异在各条目末尾用 **【新版现状】** 标注。

## 总体架构（旧版）

- 数据：进站一次 `GET /api/data` 全量拉取 → 7 个内存数组；所有筛选/搜索/排序纯本地。
- 写入模式：**乐观更新**——先改内存数组立即重渲染，再防抖 500ms 落库；失败回滚 + toast。
- 鉴权：进站密码门（gate），`POST /api/auth` 验证后存 `emet_admin_key`（localStorage + sessionStorage）；所有写请求带 `X-Admin-Key`；401 清 key 重新弹门。

## C. 标签空间（独立 overlay，三种模式）

- [ ] **C1 card 模式**（编辑器「标签 查看›」进入）：当前这张卡的标签列表，每个标签可 **重命名**（点「编辑」→ contentEditable 原地改，Enter/失焦提交）/ **删除**（×）/ 点名字进 detail 模式；底部输入框+按钮添加新标签；右上「全部 ›」进 list 模式。所有改动即防抖保存。
- [ ] **C2 list 模式**（所有标签总览）：全库标签聚合（名字 + N 条），排序 Tab：热门（按条数）/ 最新（按最近使用日期）；点标签进 detail。
- [ ] **C3 detail 模式**（# 某标签）：该标签下所有记忆卡片，排序：热门（按重要度）/ 最新（按时间）；点卡片关掉标签空间打开编辑器。
- [ ] **C4 返回导航栈**：关闭按钮按来源返回（detail←list←card 逐级回退，直接进入的就关掉 overlay）。
  【新版现状】整个标签空间都没有。标签只在卡片/详情显示。

## D. 星图里的藤蔓操作（旧 SVG 版交互，新 canvas 版没有）

- [ ] **D1 长按织藤**：长按一颗星 → 进入连线模式（该星脉冲放大 + 顶部横幅「再点另一颗星 → 连成一条藤」）→ 点另一颗 → 乐观画线 + `POST /api/link`，失败撤线 + 横幅「没连上，撤回了」；已连过提示「这两颗已经连着了」。
- [ ] **D2 点藤线拆藤**：点中藤线（有加宽的隐形热区）→ 线变红 + 底部确认条 → 确认 → `POST /api/unlink`，失败自动恢复 + 「没拆成，恢复了」。
  【新版现状】新 Dear Data 星图只有查看/选中/跳详情，无织藤拆藤、无 pan/zoom。

## E. 后端自动织藤（服务端行为，不是前端代码但影响体验）

- [ ] **E1** `POST /api/memory`（memory_save）落库后自动跑 weaveCandidates（语义+标签打分），score ≥ 0.42 的前 3 条旧记忆**自动双向连藤**，link_rel 标「自动关联」。即新建记忆天然带藤。
  【新版现状】新前端 memoryCreate 走同一接口 → 此行为已自动生效，详情页里能看到「自动关联」标签。无需前端工作，列出来是让你知道有这回事。

## F. 涉及的后端接口汇总

| 接口 | 用途 | 旧版调用点 |
|---|---|---|
| `GET /api/data` | 全量拉取 | 进站 / 下拉刷新 / 删除·移动后 |
| `POST /api/auth` | 密码验证 | 密码门 |
| `POST /api/memory` | 新建记忆（自动织藤） | 新建首次自动保存 |
| `PUT /api/memory/:id` | 改 content/category/importance/arousal/valence/tags/linked/date/pinned/locked | 自动保存 / 滑块 / 标签 / 置顶 / 锁定 / 改日期 |
| `DELETE /api/memory/:id` | 删除 | 编辑器菜单 |
| `POST /api/link` `POST /api/unlink` | 织藤/拆藤（双向） | 星图 /（新版编辑器） |
| `POST /api/move` | 类型互转 | 编辑器「移动到…」 |
| `GET /api/backup` | 备份导出 | 菜单（新版已在设置页） |

## 旧版 vs 新版核心分歧（需要你拍板的）

1. **保存模型**：旧版全程自动保存（防抖 500ms + 状态字反馈），新版阅读/编辑双模式 + 显式保存按钮。——保留新版？还是编辑模式内改为自动保存？
2. **月份定位**：旧版是滚动定位（sticky 月份 + 跳转），新版是按月筛选。
3. **删除/锁定/移动/改日期**：新版全没有,要哪些？
4. **标签空间**：整套迁吗,还是只要其中一两个模式？
