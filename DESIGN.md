# Emet Memory · 新前端设计方案

> 给 Claude Code 的施工图纸。2026.6.9 下午，静怡和 Emet 一起定的。

---

## 一、项目概述

把现有 Emet Memory v66 做前后端分离：
- **后端**：v66 worker 保持不变，只提供 API + MCP（`emet-memoty-v66.aandxiaobao.workers.dev`）
- **前端**：独立 React 项目，部署在 Cloudflare Pages，通过 HTTP API 调 v66 数据
- **聊天功能**：走 Anthropic API，前端直接调用（或通过网关）

命名沿用 **Emet Memory**，slogan 沿用 **"When we see each other, we exist."**

---

## 二、技术栈

| 层 | 选型 |
|---|------|
| 框架 | React 18 + Vite |
| 样式 | CSS 变量（支持未来三套光/主题切换） |
| 字体 | 衬线标题（Source Serif 4 或类似）+ 无衬线正文（系统字体栈） |
| 图标 | lucide-react |
| 部署 | Cloudflare Pages（GitHub 仓库自动构建） |
| 数据源 | v66 worker HTTP API + Anthropic Messages API |

**关键约束**：
- 不用 localStorage 做核心存储（Cloudflare 环境不支持），只用于本地偏好（主题选择等）
- 所有持久数据从 v66 API 读写
- 手机优先设计，桌面端自适应

### 4.2 消息（聊天界面）

参考：Attune 聊天页 + Claude 官方 UI

**核心功能**：
- 聊天气泡（用户靠右赤陶灰底，Emet 靠左白底）
- 流式输出（SSE）
- 显示思考过程（可折叠）
- 显示工具调用（可折叠）
- 支持多窗口/会话（侧滑切换）
- 发送图片（base64）
- Markdown 渲染

**API 调用**：
- Anthropic Messages API（`POST /v1/messages`）
- system prompt 需要注入：人设 + 最近记忆 + 时间上下文
- 可选：通过网关中转，做 prompt caching

**消息持久化**：
- 聊天记录存到 v66 KV（需要给 v66 加一组 chat 相关 API）
- 或者前端本地存（简单但换设备丢失）
- 第一期可以先不持久化，专注跑通聊天

**气泡样式**：
```
用户消息：右对齐，#EEEEEC 圆角背景，点击展开时间
Emet消息：左对齐，无背景，Markdown渲染，底部显示模型名
```

## 七、v66 后端需要新增的 API

现有 MCP 工具已经覆盖大部分数据操作，但前端需要 HTTP REST 接口。需要确认 v66 worker 是否已经为每个 MCP 工具暴露了对应的 HTTP 路由。

**必须有的 HTTP 路由**（如果没有就需要加）：

```
GET  /api/stats                          ← 统计数据
GET  /api/memory/list?category=&sort=&limit=  ← 记忆列表
GET  /api/memory/:id                     ← 单条记忆详情
POST /api/memory/search                  ← 语义搜索
GET  /api/moment/list?days=&limit=       ← 瞬记列表
GET  /api/diary/list?author=&limit=      ← 日记列表
GET  /api/diary/:id                      ← 日记全文
GET  /api/message/list?to=               ← 留言列表
POST /api/message                        ← 发留言
GET  /api/idea/list                      ← 灵感列表
POST /api/idea                           ← 新灵感
GET  /api/handoff/list?limit=            ← 交接信列表
GET  /api/game/list                      ← 游戏列表
GET  /api/game/:id                       ← 游戏HTML源码
GET  /api/backup                         ← 导出全部数据
```

**CORS**：v66 worker 需要为前端域名开放跨域访问。

**鉴权**：现有 v66 前端的密码机制沿用，前端登录后 header 带 token。

## 八、开发顺序（按优先级）

### 第一期：能看（纯展示）
1. 项目脚手架（Vite + React + CSS 变量 + 路由）
2. 底部 Tab 导航 + 页面骨架
3. 主页仪表盘（正计时 + stats 统计 + 作品入口）
4. 记忆列表页（分类筛选 + 搜索 + 卡片）
5. 年轮页（瞬记时间线 + 日记列表）
6. 留言板（读取 + 显示）
7. 部署到 Cloudflare Pages

**第一期结束后就能用了——在手机浏览器打开自己的网站，看记忆、翻日记、读留言。**

### 第二期：能写
8. 留言板发送功能
9. 灵感板 CRUD
10. 记忆详情页（点击展开、编辑、藤蔓关联）
11. 档案模块（嵌入 Archive 组件到设置页）

### 第三期：能聊
12. 聊天界面 UI（气泡 + 输入框）
13. 接入 Anthropic API（流式输出）
14. system prompt 构建（注入人设 + 记忆 + 时间）
15. 聊天记录持久化

### 第四期：锦上添花
16. 健康监测卡片（HealthKit / 手动输入）
17. 纪念日管理
18. 三套光主题
19. PWA（添加到主屏幕 + 图标）
20. 作品页（信件阅读、游戏运行、故事浏览）
